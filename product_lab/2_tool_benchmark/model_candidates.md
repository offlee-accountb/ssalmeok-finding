## 성능 검증 테스트 프레임워크

아이템이 정해지면, 해당 작업에 대해 아래 테스트를 수행한다.

### 테스트 설계

```markdown
# [아이템명] 모델 벤치마크

## 테스트 대상 작업
- 작업 1: [예: "HWP 문서 내용 요약"]
- 작업 2: [예: "한국어 OCR 정확도"]

## 테스트 데이터셋
- 출처:
- 규모: N건
- 언어: 한국어 / 혼합

## 평가 지표
- 정확도 (Accuracy / F1 / BLEU)
- 응답 시간 (초)
- RAM 사용량 (GB)
- 체감 품질 (1~5 수동 평가)

## 결과

| 모델 | 작업1 정확도 | 작업2 정확도 | 응답시간 | RAM | 판정 |
|------|:-----------:|:-----------:|:--------:|:---:|:----:|
| Qwen 3.5 4B | | | | | |
| Phi-4 mini | | | | | |
| ... | | | | | |

## 결론
- 최적 모델:
- 병목 지점:
- LoRA 필요 여부: → 3단계로
```

### 최소 통과 기준
- ✅ M4 Mac Mini 32GB에서 실행 가능
- ✅ 응답 시간 10초 이내 (사용자 체감)
- ✅ 한국어 정확도 80% 이상
- ✅ 연속 사용 시 메모리 누수 없음
