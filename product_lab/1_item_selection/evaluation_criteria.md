# 1단계: 아이템 평가 기준표

## 평가 축 (5점 만점)

| 기준 | 설명 | 가중치 |
|------|------|:------:|
| **시장 수요** | 지금 돈 내고 쓸 사람이 있는가? 해외에서 이미 매출 나는 유사제품이 있는가? | ×3 |
| **수익화 명확성** | 누가, 얼마에, 왜 결제하는지 명확한가? 가격 모델이 성립하는가? | ×3 |
| **1인 실현 가능성** | 혼자서 3개월 안에 MVP 가능한가? 운영도 혼자 가능한가? | ×3 |
| **구조적 접근성** | 데이터/API 접근이 합법적이고 안정적인가? | ×2 |
| **한국 시장 진입** | 한국에 같은 문제가 있는가? 한국 특화 차별점이 있는가? | ×2 |
| **경쟁 강도** | 네이버/카카오/대기업이 이미 하고 있진 않은가? | ×1 |
| **원가 구조** | API 비용 대비 마진이 나는가? 사용량이 늘면 비용도 선형으로 늘어나진 않는가? | ×1 |

## 탈락 기준 (이 중 하나라도 해당되면 제외)

- ❌ 핵심 API가 파트너/대행사 전용
- ❌ 한국 규제 위반 (금융/도박/의료)
- ❌ 네이버/카카오가 이미 무료로 제공
- ❌ 대량 유저 없으면 수익 불가 (규모의 경제)
- ❌ 혼자 운영 불가능한 구조 (대규모 고객 지원, 하드웨어 배포 등)

- 왜 돈을 내는지:

## 1인 실현 가능성 (  /5)
- 기술 스택 (API/프레임워크):
- 예상 MVP 기간:
- 혼자 CS/운영 가능 여부:

## 구조적 접근성 (  /5)

- API 의존성:

- 외국 제품이 한국에서 안 되는 이유:

## 경쟁 강도 (  /5)

- 차별화:

## 원가 구조 (  /5)

- 마진율:

## 총점:    / 75
