# 🔍 전체 아이디어 구조적 접근성 스크리닝

> 작성일: 2026-02-19  
> 기준: **핵심 데이터/기능에 구조적으로 접근 가능한가?**  
> 원칙: 구조적으로 불완전하면 C 이하

---

## 판정 기준

```
구조적 접근성이란:
  "이 서비스를 만들 때, 핵심 데이터에 합법적이고 안정적으로 접근할 수 있는가?"

✅ 완전: 공식 API 있고, 제3자 접근 열려있고, 법적 문제 없음
🟡 부분: API는 있지만 일부 핵심 채널 접근 제한
🔴 불완전: 핵심 데이터에 접근이 어렵거나 봇/크롤링에 의존
```

## 상세 판정

### 💎 SecureLocalAI KR → B (A급 잠재력)

```
핵심 데이터: 내 PC 내의 로컬 파일 (PDF, Excel, 회의록, 이메일 등)
접근 방법:
  ✅ 100% 로컬 데이터 → 유저 권한만 있으면 접근 완전 자유
  ✅ 외부 API 의존성 없음 → 보안/프라이버시 최고 수준

시장성:
  ✅ 대기업/전문직(변호사 등)의 정보유출 우려로 인한 '로컬 AI' 수요 급증
  ✅ 2026년 DDR5 RAM 성능 향상으로 GPU 없이도 쾌적한 인퍼런스(9B 4-bit) 가능
  🔴 리스크: 기업용 클라우드 AI(Copilot 등)와의 경쟁, 보안 프로그램(EDR)의 오진 가능성

등급: B (구조적 접근성 완벽, 실질적 런칭 허들 해결 시 A급 승격)
```

### ❌ Polywallets KR → F (탈락)

```
사유: 한국에서 예측시장(Polymarket 등) 자체가 규제됨.
      데이터 접근 이전에 시장 자체가 존재하지 않음.
      구조적으로 불가능.
등급: F
```

---

### ⚠️ Toolspend KR → D (비추)

```
핵심 데이터: AI/SaaS 구독 결제 내역, 사용량
접근 방법:
  ├── AI 서비스 API (OpenAI, Google AI 등) → ✅ read-only API키로 가능
  ├── 신용카드/은행 거래내역 연동 → 🔴 한국 금융 규제 (마이데이터 사업자 등록 필요)
  └── 구독 서비스 로그인 연동 → 🔴 각 서비스별 별도 연동 필요

구조적 문제:
  🔴 한국 마이데이터 사업자 등록 없이는 금융 데이터 접근 불가
  🔴 한국 기업의 AI/SaaS 지출이 아직 작음 (시장 미성숙)
  🔴 구독 관리는 네이버/카카오페이 등이 이미 부분적으로 제공

등급: D (구조적 불완전 + 시장 미성숙)
```

### 🟡 PenguinBot KR → C (외부 데이터 불필요, 그러나 다른 문제)

```
핵심 데이터: 없음 (자체 AI 에이전트)
접근성: ✅ 외부 데이터 의존 없음! LLM API만 있으면 됨

그런데:
  🔴 경쟁이 너무 심함 (ChatGPT, Claude, Gemini 자체가 이걸 함)
  🔴 2026년에 네이버 에이전트N, 카카오 카나나 출시 예정
  🔴 차별화가 거의 불가능 (범용 에이전트 시장)
  🔴 "뭘 더 잘하냐"가 없으면 존재 의미 없음

등급: C (접근성은 좋지만 경쟁/차별화 문제)
```

### ✅ Termsy KR → 구조적 접근성 최고!

```
핵심 데이터: 웹사이트 약관/이용약관/개인정보처리방침 텍스트
접근 방법:
  ✅ 약관은 100% 공개 정보 (누구나 읽을 수 있음)
  ✅ 스크래핑 필요 없음 — 유저가 보는 페이지에서 추출 (Chrome 확장)
  ✅ 법적 문제 없음 — 공개된 텍스트를 AI로 분석하는 것뿐
  ✅ API 의존 없음 — LLM API만 있으면 됨
  ✅ 제3자 허가 불필요

구조적 완전성:
  ✅ 데이터: 약관은 항상 공개
  ✅ 기술: Chrome 확장 + LLM API
  ✅ 법적: 공개 정보 분석 = 합법
  ✅ 비용: LLM API 비용만 (저렴)
```

**그런데 한국 시장에서 되나?**

```
가능성 체크:
  ✅ 한국인 99%가 약관 안 읽고 동의 → "대신 읽어주는 AI" 수요
  ✅ 개인정보 이슈 민감도 높아지는 중
  🟡 무료 Chrome 확장으로 유저 모으기는 쉬움
  🔴 근데... 돈을 어떻게 벌지? (약관 분석에 월 구독료?)
  🔴 "약관 분석했는데 그래서 뭐?" → 유저 액션이 불명확
  🔴 약관이 나빠도 그 서비스를 안 쓸 수는 없음 (카카오, 네이버...)

수익화 문제:
  개인 → 무료 기대 (약관 분석에 돈 내기 아까움)
  기업 → B2B로 "우리 약관이 괜찮은지 검사" = 가능성 있음
         하지만 법무팀이 있는 기업은 이미 자체적으로 함
```
