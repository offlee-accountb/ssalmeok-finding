## 1. Context (상황 및 배경)

나는 **1인 개발자**로, **월 $2,000 (약 200만원) 이상의 반복 수익(MRR)을 낼 수 있는 AI 기반 마이크로 SaaS** 아이템을 찾고 있다.

### 핵심 전략
- **미국/글로벌 시장**에서 **지금 막 흥행이 시작되려 하는** 아이템을 찾아라.
- 찾은 아이템을 **한국 시장에 맞게 한국화(로컬라이징)하여 내가 직접 서빙**하는 것이 최종 목표다.
- 즉, "미국에서 검증된 모델 → 한국판으로 빠르게 재구성"하는 **Fast Follower / 한국 시장 선점** 전략이다.
- 따라서 한국 시장에서도 수요가 존재하거나 곧 생길 아이템을 우선시하되, **원본 아이템의 트랙션은 반드시 미국/영어권에서 확인**되어야 한다.
- **B2C 또는 B2Prosumer**(전문직 개인, 프리랜서, 소규모 팀) 타겟을 우선 탐색하라.

### 절대 제약
- **로컬 온디바이스 AI 전용 제품은 탐색 대상에서 완전히 제외하라.** (별도 트랙에서 탐색 중)
- 클라우드 SaaS, 웹 기반, 또는 API 기반 서비스만 대상으로 하라.
- 단, "로컬 옵션을 선택 사항으로 제공"하는 하이브리드 모델은 허용한다 (핵심이 클라우드 SaaS인 경우에 한함).

## 2. Lessons Learned (이미 검토 및 탈락한 항목)

다음 항목들은 이미 검토했으나 **탈락(C~F 등급)** 처리되었다. 아래 사유와 겹치는 아이디어는 제안하지 마라.

| 아이디어 | 탈락 사유 |
|---------|---------|
| **AutoApplier** (채용 자동 지원) | 시장이 작고 경쟁이 치열. 시즈널 편차 큼 |
| **Nuanso** (광고 인사이트) | 핵심 광고 API가 대행사 전용 등 구조적 접근 불가 |
| **Termsy** (약관 분석) | 접근성 최고지만 WTP(Willingness to Pay)가 없음 |
| **Toolspend** (SaaS 지출 관리) | 금융 데이터 규제 벽 |
| **Polywallets** (예측 시장) | 사행성 규제로 한국 불가, 미국도 이미 과포화 |
| **PenguinBot** (범용 AI 에이전트) | GPT/Claude/Gemini 기본 기능과 직접 경쟁 |
| **SecureLocalAI KR** (로컬 보안 AI) | 별도 B2B 트랙으로 분리 → 이 brief에서는 제외 |

### ⚠️ 탈락 패턴에서 배운 것
위 탈락 항목들의 공통 실패 이유:
1. **"접근성은 좋은데 돈이 안 되는" 패턴** → WTP 검증이 핵심
2. **"시장은 있는데 데이터에 접근이 안 되는" 패턴** → 구조적 접근성 필수
3. **"거대 기업 기본 기능과 겹치는" 패턴** → 방어력이 없으면 탈락
4. **"로컬 100%면 결국 오픈소스로 대체되는" 패턴** → 클라우드 가치 필수

## 3. 리서치 미션 (Research Instructions)

### A. 미국 시장 초기 흥행 신호 탐지

다음 소스에서 **2025년 후반 ~ 2026년 초 사이에 등장하여 급성장 초기 신호를 보이는** AI 마이크로 SaaS를 찾아라:

- **Product Hunt**: 최근 6개월간 AI 카테고리 Top 10에 든 제품 중, 1인 또는 소규모 팀이 만든 것
- **Indie Hackers / X (Twitter)**: "first $10K MRR", "$1K MRR", "ramen profitable" 등의 키워드로 실제 수익 공개를 한 AI 도구
- **YC Launch**: 최근 2개 배치에서 AI 도구를 만든 1~3인 팀
- **BetaList / Uneed**: 최근 런칭 중에 빠르게 유저를 모으는 AI 도구
- **Reddit** (r/SaaS, r/microsaas, r/EntrepreneurRideAlong): 실제 수익을 공개하며 1인으로 운영 중인 AI SaaS
- **실제 MRR이 확인되는 사례**를 최우선으로 보고하라 (스크린샷, Stripe/Lemon Squeezy 공개 대시보드 등).

### B. 아이템 유형 필터

특별히 다음 유형에 집중하라:

1. **AI Workflow Automation**: 특정 직군의 반복 업무를 AI로 자동화하는 도구
   - 예: AI가 회의록을 자동 요약 → 태스크 생성 → 프로젝트 관리 도구에 반영
   - 예: AI가 이메일을 분류 → 자동 응답 초안 생성 → CRM 업데이트

2. **AI-Powered Content/Creative Tools**: 특정 니치의 콘텐츠 제작을 AI로 혁신하는 도구
   - 예: AI 팟캐스트 클리핑 → 쇼트폼 자동 생성
   - 예: AI 기반 특정 업종 마케팅 카피 생성기 (부동산, 이커머스 등)

3. **AI Data/Analytics for Niche**: 특정 산업/직군에 특화된 AI 분석 도구
   - 예: AI가 부동산 매물 사진을 분석하여 자동 설명 생성 + 가격 예측
   - 예: AI가 법률 문서를 분석하여 리스크 포인트 하이라이트

4. **AI-Enhanced Developer Tools**: 개발자 워크플로우에 AI를 결합한 도구
   - 예: AI 코드 리뷰봇, AI DB 쿼리 생성기, AI 문서 자동 생성기

5. **AI Agent-as-a-Service**: 특정 업무를 대신 수행하는 AI 에이전트 구독 서비스
   - 예: AI SDR (영업 아웃리치 자동화), AI 고객 지원 에이전트

### C. 한국화 적합성 추가 확인

각 아이템을 제안할 때, 다음을 반드시 함께 분석하라:
- **한국 시장에서 동일한 Pain point가 존재하는가?**
- **한국에 이미 유사 서비스가 있는가?** (있다면 그 서비스의 약점은?)
- **한국어/한국 업무 문화에 맞게 변환할 때 추가로 필요한 것은?**
- **한국 시장에서의 예상 가격대와 타겟 규모**

### D. 반드시 제외할 유형

- ❌ **범용 챗봇/어시스턴트** (GPT Wrapper 급)
- ❌ **로컬 온디바이스 전용 솔루션**
- ❌ **한국만 타겟인 처음부터 한국용 제품** (미국 트랙션 없이 한국만 노리는 것)
- ❌ **금융 데이터 접근이 필요한 제품** (마이데이터 규제)
- ❌ **규제 리스크가 높은 분야** (의료 진단, 법적 자문, 금융 투자 조언 등에서 직접적 의사결정을 내리는 것)

## 4. 아이디어 평가 매트릭스 (Evaluation Matrix)

각 제안 아이디어는 다음 6가지 기준에 따라 **각 10점 만점, 총 60점**으로 점수를 매기고 보고하라.

| 항목 | 기준 | 높은 점수 예시 |
|:---:|---|---|
| **WTP 강도** | 타겟이 실제로 매달 돈을 내고 있거나, 낼 만한 고통(Pain)이 명확한가? | 이미 유사 도구에 $20+/월을 지불하는 유저 존재 |
| **구조적 접근성** | 핵심 데이터/기능에 합법적이고 안정적으로 접근 가능한가? | 공개 API, 공개 데이터, 유저 제공 데이터만으로 동작 |
| **방어력 (Moat)** | GPT/Claude/Gemini 기본 기능이나 거대 기업 제품으로 쉽게 대체 불가능한가? | 특정 도메인 데이터 파이프라인, 커스텀 워크플로우, 네트워크 효과 |
| **1인 운영 가능성** | 개발~운영~고객 지원을 1인이 감당 가능한가? 인프라 비용이 합리적인가? | 셀프서브 SaaS, 낮은 인프라 비용, 고객 지원 부담 적음 |
| **한국화 적합성** | 한국 시장에 한국화하여 서빙했을 때 충분한 수요가 있는가? | 한국에 동일 Pain point 존재, 유사 서비스가 아직 없거나 약함 |
| **타이밍** | 지금이 진입 적기인가? 너무 이르거나 너무 늦지 않은가? | 미국에서 트랙션 시작, 한국엔 아직 안 들어옴 |

**A급 기준: 48점 이상 / B급: 36~47점 / C급 이하: 35점 이하**

## 5. 탐지해야 할 "초기 흥행 신호" 정의

단순히 아이디어가 좋아 보이는 것이 아니라, **실제 시장에서 트랙션이 시작되고 있는 증거**를 우선시하라.

| 신호 | 구체적 지표 |
|------|-----------|
| **실제 수익** | 공개된 MRR $1K 이상, Stripe/Lemon Squeezy 대시보드 |
| **유저 성장** | Product Hunt 500+ upvote, 웨이팅리스트 1,000+명 |
| **소셜 버즈** | X/Twitter에서 관련 키워드 멘션 급증, Reddit 긍정 리뷰 |
| **카피캣 등장** | 유사 제품이 2~3개 이상 나오기 시작 (시장 validation) |
| **VC/엔젤 관심** | 소규모 시드 라운드 소식, YC 배치 선정 |
| **"How I built" 콘텐츠** | 개발자가 빌딩 과정을 공유하며 주목 받는 경우 |

## 6. 최종 출력 요구사항 (Final Deliverables)

리서치 결과로 **최소 5개 이상의 후보 아이템**을 제안하되, 각 아이템마다 다음을 포함하라:

### 각 아이템별 필수 포맷:

```
### [아이템 명칭] — [한 줄 요약]

**초기 흥행 신호:** [실제 트랙션 증거 — MRR, 유저 수, 소셜 반응 등]
**벤치마크 제품:** [이미 존재하는 유사 제품/경쟁사 + 그들의 약점]

**타겟 페르소나:** 누가 이걸 쓰고, 왜 돈을 내는가?
**핵심 기능 시나리오:** 어떤 AI 기술을 써서 어떤 액션을 하는가?
**구조적 접근성:** 어떤 데이터를 쓰고, 그 데이터에 어떻게 접근하는가?

**비즈니스 모델:** 
- 가격대: $XX/월
- 프리미엄/프리티어 전략
- 1000명 유료 유저 시 예상 MRR

**한국화 전략:**
- 한국 시장 Pain point 존재 여부
- 한국 내 유사 서비스 현황
- 한국화 시 추가 필요 요소
- 한국 시장 예상 가격대: 월 X만원
- 한국 타겟 규모: X만명

**평가 점수:**
| WTP | 접근성 | 방어력 | 1인운영 | 한국화 | 타이밍 | 총점 |
|-----|-------|-------|--------|-------|-------|------|
| X/10 | X/10 | X/10 | X/10 | X/10 | X/10 | XX/60 |

**내가 만든다면 (MVP 2개월 기준):**
- 기술 스택: ...
- Week 1-2: ...
- Week 3-4: ...
- Week 5-8: ...

**이걸 하지 말아야 할 이유 (Devil's Advocate):**
- ...
```

### 추가 요구사항:
- 5개 이상 후보 중 **1~2개는 "남들이 아직 잘 안 보고 있지만 분명히 올 것"인 아이템**을 포함하라 (contrarian pick).
- 각 아이템에 대해 **"이걸 하지 말아야 할 이유" 1가지**도 반드시 포함하라 (Devil's Advocate).
