# 📋 쌀먹 파인딩 — 상위 전략 지침

> 최종 수정: 2026-03-01  
> 이 문서는 파인딩 프로그램의 **전략적 방향과 제약**을 정의한다.  
> 실제 ChatGPT Deep Research 프롬프트(`2_prompt.md`)는 이 지침을 따른다.  
> **이 문서는 분기에 1~2회 정도만 수정**하며, 방향이 크게 바뀔 때만 건드린다.

---

## 1. 나는 누구인가

- **1인 개발자**. 풀타임 직장이 있고, 사이드 프로젝트로 수익화를 노린다.
- 기술 스택: Python, JS/TS, Next.js, FastAPI 등 풀스택 가능. AI/LLM API 사용 경험 있음.
- AI/스마트제조 도메인 경험 있음 (정부 프로젝트 수행 경력).
- 1인 운영이 핵심 제약 — 고객 지원, 인프라 관리, 마케팅 모두 혼자 감당 가능해야 함.

---

## 2. 목표

**월 200만원($2,000) 이상의 반복 수익(MRR)**을 낼 수 있는 AI 기반 마이크로 SaaS 아이템을 찾아, 한국 시장에서 서빙한다.

### 핵심 전략: Fast Follower → 한국 선점

```
미국에서 이미 트랙션이 검증된 AI 마이크로 SaaS
    │
    ▼  한국화 (로컬라이징)
    │  - 한국어 UI/UX
    │  - 한국 시장 특유의 니즈 반영
    │  - 한국 결제/인프라 연동
    │
    ▼
한국 시장에 Fast Follower로 선점 서빙
```

- **"한국에서만 통하는" 아이디어를 처음부터 만드는 게 아니다.**
- **미국에서 먹히고 있는 걸 찾아서, 한국판으로 빠르게 재구성한다.**
- 원본의 트랙션이 미국/영어권에서 확인된 것만 대상으로 한다.

---

## 3. 탐색 트랙 분리

| 트랙 | 설명 | 이 지침에서 |
|------|------|:---:|
| **트랙 A: 클라우드 SaaS** | 미국발 검증 모델 → 한국화 서빙. B2C/B2Prosumer | ✅ **이 트랙** |
| **트랙 B: 로컬 B2B** | 로컬 온디바이스 AI 기반 기업용 솔루션 | ❌ 별도 탐색 |

- 트랙 B(로컬 B2B)는 이미 구체화가 진행 중이므로 이 파인딩 프로그램에서는 **완전히 제외**한다.
- 이 지침과 하위 프롬프트는 **트랙 A(클라우드 SaaS)에만 집중**한다.

---

## 4. 절대 제약 (변하지 않는 것)

1. **로컬 온디바이스 AI 전용 제품은 제외.** (트랙 B로 분리됨)
2. **클라우드 SaaS / 웹 기반 / API 기반 서비스만 대상.**
3. **로컬 옵션이 "선택 사항"인 하이브리드는 허용** (핵심이 클라우드 SaaS일 때).
4. **규제 리스크가 높은 분야 직접 의사결정 제외** (의료 진단, 법적 자문, 금융 투자 조언 등).
5. **범용 챗봇/GPT Wrapper는 제외.** 거대 기업 기본 기능과 직접 경쟁하는 것은 안 됨.
6. **금융 데이터 접근 필요 제품 제외.** (마이데이터 규제)

## 5. 누적 학습 — 탈락 아이디어 & 실패 패턴

### 탈락 항목 (C~F 등급)

> ⚠️ 새로운 탈락이 확인될 때마다 여기에 추가한다.

| # | 아이디어 | 탈락 사유 | 등급 |
|---|---------|---------|:---:|
| 1 | AutoApplier (채용 자동 지원) | 시장 작음, 경쟁 치열, 시즈널 편차 | C |
| 2 | Nuanso (광고 인사이트) | 핵심 광고 API가 대행사 전용 (구조적 접근 불가) | C |
| 3 | Termsy (약관 분석) | 접근성 최고지만 WTP 없음 | C |
| 4 | Toolspend (SaaS 지출 관리) | 금융 데이터 규제 벽 | D |
| 5 | Polywallets (예측 시장) | 사행성 규제로 한국 불가, 미국도 과포화 | F |
| 6 | PenguinBot (범용 AI 에이전트) | GPT/Claude/Gemini 기본 기능과 직접 경쟁 | C |
| 7 | SecureLocalAI KR (로컬 보안 AI) | 트랙 B로 분리 (이 트랙에서 제외) | — |

### 탈락 패턴 (반복하지 말 것)

```
패턴 1: "접근성은 좋은데 돈이 안 됨"
  → Termsy 케이스. WTP 검증이 최우선.

패턴 2: "시장은 있는데 데이터에 접근이 안 됨"
  → Nuanso 케이스. 구조적 접근성 사전 확인 필수.

패턴 3: "거대 기업 기본 기능과 겹침"
  → PenguinBot 케이스. 방어력(Moat)이 없으면 탈락.

패턴 4: "로컬 100%면 오픈소스로 대체됨"
  → SecureLocalAI 케이스. 클라우드 가치가 필수. (이 트랙에선 애초에 제외)
```

## 6. 성공 기준

아이템이 A급으로 인정받으려면 다음을 **모두** 충족해야 한다:

| 기준 | 최소 요건 |
|------|---------|
| **WTP 강도** | 유사 도구에 $10+/월 지불하는 유저가 실제로 존재 |
| **구조적 접근성** | 공개 API 또는 유저 제공 데이터만으로 동작 가능 |
| **방어력** | GPT/Claude 기본 기능으로 완전 대체 불가능 |
| **1인 운영 가능** | 셀프서브 SaaS, 고객 지원 부담 적음, 인프라 비용 합리적 |
| **한국 시장 적용 가능** | 한국화했을 때 수요가 존재하거나 곧 생길 것 |

## 7. 프롬프트 운영 원칙

- `2_prompt.md`는 이 전략 지침을 따르되, **ChatGPT에게 줄 실행용 프롬프트**로 구성한다.
- 매 Deep Research 실행 후 결과를 반영하여 `2_prompt.md`를 **점진적으로 업데이트**한다.
  - 새 탈락 아이디어 → 이 문서의 5장에 추가
  - 탐색 방향 미세 조정 → `2_prompt.md`의 필터/유형 조정
- **이 문서(1_strategy.md)는 전략 방향이 크게 바뀔 때만 수정한다.**
